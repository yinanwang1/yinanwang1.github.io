最近10年有余的工作总结，看看都做了些啥呢。在同一家公司，伴随着公司一起成长，见证了公司的起起落落。在公司需要我的时候，我都挺身而出，希望公司茁壮成长。并加倍地努力，一直保持学习，把自己培养成六边形战士。一直在进步，一直在改变。

一、 59

2015年加入做零食盒子的业务，当年业务发展很快，似乎都是一个月业绩就翻一番，然后人员招得也很多，技术团队150人吧。我负责当时的移动端，有12人，也参与iOS的开发工作。APP更新周期差不多都是一周。创业开始阶段，思想很活跃，新想法一直在冒出。

当时app的名称为59store，业务有夜猫子，云打印，饮料。不同业务模块有不同的开发人员维护。代码管理使用git，大家都适应多人开发。同时为了代码更好的管理，将代码进行分模块管理，每个模块创建一个git仓库，然后通过版本号来依赖。

使用友盟来进行崩溃日志采集和埋点记录。并且会在提测前，进行code review，并分享下设计思路。从而保证代码的健壮性和业务的稳定性。

二、 骑电

2016年7月份，公司内部孵化成立了骑电。全体人员都转入骑电，从0开始思考、预演、设计电单车的租赁业务，主要涉及到电单车的使用，调度和换电，3个大模块。硬件上主要是中控系统的开发。

我是继续担任移动端的负责人。

1. android

一直是从事iOS的开发，觉得iOS是未来，深入研究iOS底层学习，成为行业的专家是我的目标。

没想到2017年共享单车引来政策的大调整，业务量大量的萎缩，导致公司缩减规模。Android端开发进展拖了整体进度。想着我抽出时间来调节下Android的进度，基础的功能开发我也参加下。同时也可以更加保证iOS和Android的功能一致性。

《疯狂Android讲义》入门，接着开始《第一行代码 2》开始学习，将书本中知识点都自己实现了一遍。后来《第一行代码 Android3》出版了，又买了一本，再学习了一遍，讲到的最佳实践都在项目中直接使用。接着学了《Android开发艺术探索》，里面的理论基础讲得很多，对Android的底层有了更多地了解。

感觉自己对Android有了全面了解后，觉得是时候出手了。从修改bug开始，参与一些小需求的开发，慢慢地几个版本结束，大致已经熟悉Android的开发流程，网络请求、页面绘制等。

此时新版本开发时，Android和iOS我可以一人搞定，并且可以保证逻辑一致。

2. 担任运营端的负责人。

2018年企业从共享自行车开始涉足共享电动车，供应链开始完善，生产车辆越来越快，投放的车辆也越来越多。运营成为了公司中重要部门，但需要从0开始摸索。任命我为运营端的负责人，配备了1个产品经理，1个测试人员，1个UI设计师，2个后端研发人员，1个android开发人员和我iOS开发人员。

研发采用敏捷开发，将运营中紧急的任务快速迭代。收集领导层、客服、运营人员、用户等的需求，按优先级和工作量进行排期。周一进行PRD（产品需求文档）的评审，会后研发人员进行需求的分解并确定方案，同时测试人员进行测试文档的编写。周二早上过一遍测试用例，大家都同步需求细节。然后开始开发，争取在周三进行提测，并在周四测试完成，周五前发布到生产环境。

记得当时公司有一个全职的运维人员，负责公司服务器管理和版本发布。我们运营端发布版本是最密集的。

2018年的一整年，发布了将近50个版本，将共享电单车需要用到的运营功能都开发完成。主要涉及到功能有：故障车打捞，批量调度，单车换电，车辆质检，仓库管理，还车点管理等。

3. python

很是一个巧合的机会，需要分析订单数量，研究发现使用python是最方便。开始全面的了解python，下载工具，搭建环境，买了2本python的入门书，看了几本电子书后。感觉到自己已经有能力参与python开发了。也是巧合，公司的单车模拟器需要维护，而我又会python。

熟悉了模拟器使用的框架，单车的通讯协议后，问题的修复和新需求的开发都由我跟进。

同时，数据分析，日志分析，导出数据，重复性工作，统统都编写成python脚本。然后把python文件给产品，可以一直重复使用了。大大地提交了工作效率。

同时为了保持python的熟悉度，使用python来编写算法。

4. 前端、小程序

2021年，前端岗位空缺了一个月，我也是硬着头皮顶上了。先学习了ant-design库，将所有的组件和demo都看了一遍。再学习了wepy库，一个第三方项目用来打包微信和支付宝小程序。同时也是恶补了js和css的语法。并把微信小程序官方文档的小程序部分都学习了一遍，所有接口都了解是做什么用的。

等学习完成后，开始放手进行bug的修改。同时对公司的ERP系统，进行第三方库的升级。在升级的过程中，提高了业务和技术的水平。将所有的第三方库都升级到最新，自己反复测试后，提测并发布。

小程序修复bug中，有一个mqtt连接失败的问题，困恼了很久。因为mqtt库使用的是第三方的，然后再二次开发来支持微信和支付宝。花了一个星期，对mqtt文件进行逐行的分析和研究，一步一步调试重现，最终找到问题的原因，将问题修复。

5. flutter

作为一个iOSer，开始android开发后，每次在编写Android代码时，脑子中总是出现iOS的实现方案。而在iOS开发过程中，又想着Android代码是否有功能遗漏。虽然使用了Xmind来管理功能点，将大任务分解成小任务，然后同时开发iOS和Android，来提升效率。

一直关注着跨平台的技术，flutter自学了2年，学了几本flutter的入门和精通，也编写了LearnPython的demo。对widget和state了然如胸后，在项目中开始加入flutter。新功能开发的时候，优先考虑flutter的使用。效率提升4倍吧，因为发现使用flutter开发比直接用native开发，效率提升2倍。然后又省下一端的开发。直接4倍的提升。突然发现我的iOS和Android开发效率高得离谱。

在使用flutter中碰到扫码慢的问题，使用得mobile_scanner三方库。深入了解后，减少扫码的类型，指定识别的区域，优化数据的解析等等。Android端基本可以满足运营人员的要求了。iOS端还是不停地有人抱怨，说扫描3秒后识别不出，相机倾斜一点都不行。继续从mobile_scanner的iOS端代码开始查找解决方案，发现没有使用iOS自带的AVCaptureMetadataOutput来实现二维码的识别。而iOS自带的效率很高，so直接修改源码，并达到了预期的效果。

https://github.com/juliansteenbakker/mobile_scanner/issues/569

此方案还被作者采纳，在新版本中使用了此方法。

6. java

运营端的后端研发人员离职后，发现运营业务问题分析需要从代码上来分析。那就申请后端的代码权限吧，然后把Intellij IDEA装起来。碰到不懂的知识点，深入熟悉。接着从常用注解学起，spring boot、rabbit mq、netty、redis、kafka、mybatis plus等常用框架的学习，多线程、互斥锁的研究。

运营端的后端服务dispatcher从controller层开始入手，将代码执行的过程整理清楚并做好笔记。如果是完整的业务，按业务逻辑编写后整理成笔记。之后查问题、新功能开发，上手就非常地容易了。

我一个人就可以撑起整个运营业务的线上问题定位，bug的修复，新功能地开发。编写好后端java代码后，编写API的接口文档，然后运营端iOS和Android功能使用flutter对接接口，最后自己联调功能。效率比多个人快很多。

在出国游玩的时候，用英语购物问路都很自信。在工作中，查看英文文档有利于对新技术的了解。

四、 算法

算法和设计模式，也是学了忘，忘了有学。在项目空隙，都会再拿起来研究一番。对代码逻辑的提升非常的明显，并且有利于对问题的从不同角度来思考解决方案。一直在Leetcode算法练习上进行练习。

碰到轻松能解决的算法题，让人自信爆满。碰到有难度的算法题，让人兴奋、大脑飞速运转，花费时间最后解法，真的是酣畅淋漓啊，忍不住要分享。碰到难度太大，尝试多次还是失败后，会失落，但是在学习最优解后，自己再默默地再实现一边，把解法成为自己的能力。感觉能力大涨的痛快。

七、 小程序

工作中用到了编写小程序，但是用的是wepy框架，太古老了，而且已经停止了维护。升级或替换的可能性不大，所以一直想用taro写一个小程序。“弥可学字”，从0到1开始搞起。当时女儿刚开始学汉字，猜字谜是很好的方法，那就做一个吧。写界面花了一天时间搞定了，但是字谜从哪里来。思路是找出一年级到六年级要学的汉字，然后为每一个汉字编写字谜。

花了很大的力气去搜集字谜，可是进度缓慢啊。最后还是放弃自己一个一个搜，而是直接编写了一个python脚本，从网络上把字谜获取，然后输出json字符串。直接放到小程序中。（计划放到服务器，通过接口获取）

弥可学字-哔哩哔哩

界面录制了一个视频放到了B站。“番茄钟”、“24点”和“日地月轨迹”这3个是用了codex这个AI agent来完成的。由AI来完成主要功能，然后再手动修改一些代码来完成细节。
